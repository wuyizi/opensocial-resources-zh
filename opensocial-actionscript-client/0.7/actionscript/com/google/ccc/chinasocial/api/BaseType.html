Classpublic class BaseType
SubclassesActivity, Address, Email, Message, Name, Organization, Person, Phone, Url

Base type wrapped for all opensocial data structure. It's an abstract class.

All these types will have a common property set: "fields" and some getField related methods.
Types like Person, Activity, Url, Email are the extended from this type. Only the Collection type is not extended from this type because Collection does not has "fields" property set.

NOTE: The BaseType in As-side is not straight related to the flashAPI.SimpleType class in Js-side.
In Js-side no inheritance is introduced. The flashAPI.SimpleType stands for the simple opensocial data structure like Address, Email, etc..
But in As-side, Address or Email types are simply extended from BaseType.

Here is the graph to represent the relation: 
    As-side                 Js-side                   opensocial namespace
   ======================================================================== 
    BaseType(abstract)
      + Address             flashApi.SimpleType       opensocial.Address
      + Email               flashApi.SimpleType       opensocial.Email
      + Name                flashApi.SimpleType       opensocial.Name
      + Orgnaization        flashApi.SimpleType       opensocial.Orgnaization
      + Phone               flashApi.SimpleType       opensocial.Phone
      + Url                 flashApi.SimpleType       opensocial.Url
      + Activity            flashApi.Activity         opensocial.Activity
      + Person              flashApi.Person           opensocial.Person
      + Message             flashApi.Message          opensocial.Message
    Collection              flashApi.Collection       opensocial.Collection
